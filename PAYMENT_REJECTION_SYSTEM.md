# نظام رفض الإيصالات ومنع متابعة الطلب

## الوصف
تم تطبيق نظام يمنع متابعة الطلب عند رفض إيصال الدفع البنكي حتى يتم رفع إيصال جديد من العميل.

## الميزات المضافة

### 1. منع متابعة الطلب عند رفض الإيصال
- عند رفض إيصال الدفع، لا يمكن تحديث حالة الطلب إلى أي حالة متقدمة
- يظهر خطأ واضح للمتجر عند محاولة متابعة الطلب
- الاستثناء الوحيد هو إلغاء الطلب (cancelled)

### 2. رفع إيصال جديد بعد الرفض
- العميل يمكنه رفع إيصال جديد عبر تطبيق تتبع الطلبات
- يتم إشعار المتجر تلقائياً عند رفع إيصال جديد
- يتم تسجيل العملية في تاريخ الطلب

### 3. واجهة محسنة للمتاجر
- خيارات متعددة لرفض الإيصال مع الأسباب:
  - إيصال غير واضح
  - مبلغ خاطئ  
  - إيصال مزيف
- رسائل تحذيرية واضحة عند رفض الإيصال

### 4. واجهة محسنة للعملاء
- إشعار واضح عند رفض الإيصال
- إمكانية رفع إيصال جديد بسهولة
- عرض حالة الدفع في تتبع الطلبات

## الملفات المحدثة

### Backend
- `backend/routes/orders.js`: إضافة منطق منع متابعة الطلب + endpoint للإيصال الجديد
- `backend/models/Order.js`: التأكد من وجود حالة `rejected`

### Frontend  
- `src/components/PaymentRejectionHandler.js`: مكون جديد لمعالجة رفض الإيصال
- `src/components/OrderTracking.js`: عرض حالة رفض الإيصال وإمكانية رفع إيصال جديد
- `src/components/OrderDetailView.js`: خيارات رفض الإيصال مع الأسباب
- `src/components/ShopOrderManagement.js`: تحديثات طفيفة

## API Endpoints الجديدة

### PUT /orders/:orderId/new-receipt
رفع إيصال جديد بعد الرفض
```json
{
  "receiptUrl": "/uploads/new-receipt.jpg"
}
```

### PUT /orders/:orderId/payment-status (محدث)
تحديث حالة الدفع مع إمكانية إضافة ملاحظة
```json
{
  "paymentStatus": "rejected",
  "note": "الإيصال غير واضح"
}
```

## تدفق العمل

1. **العميل يرفع إيصال**: `paymentStatus = 'receipt_uploaded'`
2. **المتجر يرفض الإيصال**: `paymentStatus = 'rejected'` + ملاحظة
3. **منع متابعة الطلب**: أي محاولة لتحديث حالة الطلب تفشل
4. **العميل يرفع إيصال جديد**: `paymentStatus = 'receipt_uploaded'` مرة أخرى
5. **المتجر يوافق**: `paymentStatus = 'confirmed'`
6. **يمكن متابعة الطلب**: تحديث الحالة يعمل بشكل طبيعي
